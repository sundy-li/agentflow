# CLI Board Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a CLI command that reads the local SQLite database and prints the current task board grouped by state.

**Architecture:** Create a separate `app/cli.py` entry point that loads config, selects the active repo, reads tasks through `Repository`, groups them by workflow state, and renders a plain-text table per state. Keep the implementation independent from the HTTP board API so the command works even when the web service is not running.

**Tech Stack:** Python 3.10+, `argparse`, SQLite repository layer, pytest.
